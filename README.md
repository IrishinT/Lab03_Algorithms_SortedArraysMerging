# Лабораторная работа: Слияние отсортированных массивов

Данная лабораторная работа посвящена исследованию и сравнению алгоритмов слияния двух отсортированных массивов. В рамках работы реализованы два подхода к слиянию: классический алгоритм с созданием нового массива и алгоритм слияния "на месте" (in-place) с использованием циклических сдвигов.

## Структура проекта

Проект состоит из следующих компонентов:

### 1. Библиотека алгоритмов (Service)

- **MergeAlgorithms** - статический класс, содержащий реализацию алгоритмов слияния:
  - `SimpleMerge` - простое слияние с созданием нового массива
  - `InPlaceMerge` - слияние "на месте" с использованием циклических сдвигов

- **ArrayGenerator** - генератор отсортированных массивов со случайными значениями

- **MemoryCalculator** - калькулятор для оценки используемой памяти

### 2. Модуль валидации (Validation)

- **ValidationHelper** - вспомогательный класс для проверки корректности входных данных и отсортированности массивов

### 3. Графический интерфейс (Lab03_Algorithms_SortedArraysMerging)

- Windows Forms приложение для визуализации работы алгоритмов
- Возможность задания размеров массивов и диапазона значений
- Отображение исходных массивов и результатов слияния
- Таблица с метриками производительности

### 4. Модульные тесты (Tests)

- Набор тестов для проверки корректности работы обоих алгоритмов
- Тестирование на различных входных данных, включая граничные случаи

## Алгоритмы слияния

### Простое слияние (SimpleMerge)

Классический алгоритм слияния, который создает новый массив и последовательно заполняет его элементами из двух исходных массивов в отсортированном порядке.

**Временная сложность:** O(n + m), где n и m - размеры исходных массивов

**Пространственная сложность:** O(n + m) - требуется дополнительная память для результирующего массива

### Слияние "на месте" (InPlaceMerge)

Альтернативный алгоритм, который сначала объединяет элементы обоих массивов в один общий массив, а затем выполняет сортировку слиянием с использованием циклических сдвигов элементов.

**Временная сложность:** O(n * m) в худшем случае (из-за циклических сдвигов)

**Пространственная сложность:** O(n + m) - также требует создания общего массива (в текущей реализации)

## Системные требования

- .NET 5.0 или выше
- Windows Forms
- Visual Studio 2019/2022 (рекомендуется)

## Инструкция по использованию

1. Запустите приложение
2. Укажите размеры первого и второго массивов (от 10 до 5000)
3. Задайте минимальное и максимальное значения для генерации элементов
4. Нажмите "Заполнить массив" для генерации отсортированных массивов
5. Нажмите "Слияние" для выполнения обоих алгоритмов
6. Просмотрите результаты в таблице и списках справа

## Метрики производительности

Для каждого алгоритма отображаются следующие метрики:
- Время выполнения (в миллисекундах)
- Количество выполненных элементарных операций
- Использование памяти (в байтах)

## Тестирование

Проект содержит модульные тесты, проверяющие:
- Корректность работы алгоритмов на различных наборах данных
- Обработку граничных случаев (пустые массивы, массивы с одним элементом)
- Сравнение результатов обоих алгоритмов
- Работу с большими массивами

Для запуска тестов используйте стандартные средства Visual Studio (Test Explorer).
